You should read the content of the Description tab before reading the information below.

Adding class representing the lesson

To add a class representing lesson you should extend already prepared adapter for Facelets view technology - FaceletLessonAdapter.

Adding view files for the lesson

For lesson with ExampleLesson view name, the path for view file is \src\main\webapp\facelets\lesson\ExampleLesson\index.html.
To create view for this technology it is enough to create index.xhtml file with the following content: 
        
             <?xml version="1.0" encoding="UTF-8"?>
             <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
            "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
             <html xmlns="http://www.w3.org/1999/xhtml"
                  xmlns:ui="http://java.sun.com/jsf/facelets">

             <ui:composition template="../../index.xhtml">
                 <ui:define name="content">

                     <!-- lesson UI here -->

                 </ui:define>
             </ui:composition>
             </html>
Java class connected with this UI file should have the following content: 
        
            public class ExampleLesson extends Composite implements GWTLesson {

                interface ExampleLessonUiBinder extends UiBinder<Widget, ExampleLesson> {}

                private static ExampleLessonUiBinder uiBinder = GWT.create(ExampleLessonUiBinder.class);

                public ExampleLesson() {
                    initWidget(uiBinder.createAndBindUi(this));

                    /* lesson content */
                }
            }
HTML files for the security problem description and solution
For lesson with ExampleLesson view name, the path for solution.html and problem.html files is \src\main\webapp\Facelets\lesson\ExampleLesson\.
