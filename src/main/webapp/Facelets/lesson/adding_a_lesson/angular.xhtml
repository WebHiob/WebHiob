Adding class representing the lesson

To add a class representing lesson you should extend already prepared adapter for AngularJS view technology - AngularJSLessonAdapter.

Adding view files for the lesson

For lesson with ExampleLesson view name, the view file path is \src\main\webapp\angularjs\lesson\ExampleLesson\index.html.
To create view in this technology designer should provide:
  • AngularJS controller
  • index.html file
AngularJS controller has to have the name starting with view name of the class representing lesson and ending with Controller. Designer can put this controller in already existing \src\main\webapp\angularjs\js\main.js file or in new file (\src\main\webapp\angularjs\js\controller is recommended path), however then designer has to add additional script tag with reference to this file in the end of head tag in the \src\main\webapp\angularjs\index.html file. Example of empty controller for the lesson with ExampleLesson view name is shown below: 
        
            app.controller('ExampleLessonController', function () {
            /* body of controller */
            }
Lesson file ( index.html) has to have div tag with ng-controller directive. Example of empty lesson file: <div ng-controller="ExampleLessonController"> </div>
HTML files for the security problem description and solution
For lesson with ExampleLesson view name, the path for solution.html and problem.html files is \src\main\webapp\angularjs\lesson\ExampleLesson\.
