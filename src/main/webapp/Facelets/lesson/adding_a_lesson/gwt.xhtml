You should read the content of the Description tab before reading the information below.

Adding class representing the lesson

To add a class representing lesson you should extend already prepared adapter for GWT view technology - GWTLessonAdapter.

Adding view files for the lesson

For lesson with ExampleLesson view name, the view files path are
  • \src\main\java\com\webhiob\gwt\lesson\ExampleLesson.java
  • \src\main\java\com\webhiob\gwt\lesson\ExampleLesson.ui.xml
To create view in this technology you should provide:
  • XML file representing GWT UI
  • Java class extending Composite class and implementing GWTLesson interface connected with XML UI file
XML file (ExampleLesson.ui.xml) should have the following content: 
    
     <ui:UiBinder xmlns:ui='urn:ui:com.google.gwt.uibinder'
                 xmlns:g="urn:import:com.google.gwt.user.client.ui">
         <g:VerticalPanel>

             <!-- lesson UI here -->

         </g:VerticalPanel>
     </ui:UiBinder>
HTML files for the security problem description and solution
For lesson with ExampleLesson view name, the path for solution.html and problem.html files is \src\main\webapp\GWT\lesson\ExampleLesson\.
To take advantage of the code syntax highlighter it is required to provide HTML page with the following content: 
        

            <html>
            <head>
                <link rel="stylesheet" href="../../../shared/highlight/styles/default.css"/>
                <script src="../../../shared/highlight/highlight.pack.js"></script>
                <script>hljs.initHighlightingOnLoad();</script>
            </head>
            <body>
            <!-- security problem description or solution -->
            </body>
            </html>
