Spring MVC is prepared for this security issue and provides the following methods for the data binder: setAllowedFields and setDisallowedFields. It is always recommended to provide a white-list validation, however if only one field should not be bound, then it is more convenient to use the setDisallowedFields method. In this lessons only the login filed should not be bound, therefore the latter method is a good choice. The code providing the security is shown below and should be used in the DataSubmissionToNonEditableFieldsController class: 
      
         @InitBinder
         public void initBinder(WebDataBinder binder) {
            binder.setDisallowedFields("login");
         }
