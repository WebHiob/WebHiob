  • 3.0.0 to 3.0.5
  • 2.5.0 to 2.5.6.SEC02 (community releases)
  • 2.5.0 to 2.5.7.SR01 (subscription customers)

To fix this security issue the Spring framework has introduced a new context parameter - springJspExpressionSupport, which set to false disable the double resolving issue. However this parameter is only available in the following versions [30]:
  • 3.0.6 onwards
  • 2.5.6.SEC03 onwards (community releases)
  • 2.5.7.SR02 (subscription customers)
For others versions it is recommended to provide an input validation to avoid that user's data will be interpreted as another EL expression.
