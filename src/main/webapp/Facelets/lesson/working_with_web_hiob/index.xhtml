User interface of WebHiob is self-explanatory. Each lesson contains three tabs: lesson, security problem and solution. Each tab's content provides description, therefore you should not have any problem with using WebHiob.
To be able to take advantage of all lessons it is recommended to install Zed Attack Proxy (or similar tool allowing to intercept HTTP requests and responses).

This tab contains the lesson, whose name is displayed above of the name of the tab.

Aim of the lessons

On successful completion of each lesson you will be able to:
  • test web applications for the specified attack
  • protect web applications from the specified attacks
The learning steps:
  • 1. Read the content of the lesson's tab and try to find security problem and solution on your own.
  • 2. If you will not be able to find the security problem, you should read the content of 'Security problem' tab, perform the attack and localize each described vulnerable code fragment in WebHiob and ensure that you understand why the attack is possible. If you were able to find security problem, you should also read this content to ensure that you have understood everything right. This content could also contain more generic description, therefore anyway it is valuable to read it.
  • 3. If you will not be able to find solution how to protect WebHiob, you should read the content of 'Solution' tab, perform the solution in WebHiob and ensure that attack is no more possible. If you were able to find security problem, you should also read this content to ensure that you have done everything right. This content could also contain more generic description, therefore anyway it is valuable to read it.
To avoid repeating in each description of how to complete the lesson, the part that the user should find vulnerable code fragment and then protect WebHiob, is omitted.

WARNING 1

While running WebHiob your machine will be vulnerable to attack, therefore you should disconnect from the Internet while using it.
